# **A. Project Goal**

Build the backend system that captures, scores, classifies, and routes every lead generated by True North’s content engine. The content creation and repurposing pipeline (Stages 1–2) already exist. This build covers Stage 3 only: the post-click experience from landing page through CRM automation.

The system must handle three distinct lead magnets, score leads consistently regardless of entry point, classify them into hot/warm/cold, and route each classification into the correct follow-up workflow.

# **B. Funnel Architecture**

## **B1. Quiz Funnel (/quiz)**

**Purpose:** Qualify families who are unsure whether it is time for assisted living.

**CTA Promise:** “Take the 2-minute assisted living readiness assessment.”

**Flow:** Landing page → survey/assessment → results page → CRM intake → score → classify → route

## **B2. Calculator Funnel (/calculator)**

**Purpose:** Capture higher-intent families trying to understand affordability and funding options.

**CTA Promise:** “Estimate what assisted living may cost and how families typically pay.”

**Flow:** Landing page → calculator form → results page with cost range → CRM intake → score → classify → route

## **B3. Guide Funnel (/guide)**

**Purpose:** Capture early-stage leads who want education before taking action.

**CTA Promise:** “Download the free Assisted Living Planning Guide.”

**Flow:** Landing page → opt-in form → thank-you page with PDF link → CRM intake → score → classify → route to nurture

## **B4. Booking Page (/book)**

**Purpose:** Destination for hot leads. Consultation booking via GHL calendar or Calendly embed.

**Note:** This may be a standalone page or embedded on thank-you/results pages for hot-classified leads.

# **E. Guide Funnel — Inputs**

The guide funnel captures early-stage leads. The original spec collected only name/email/phone, which meant guide leads had almost no scoring data and would always classify cold. Two qualifying questions have been added to allow minimum viable scoring.

### **Opt-In Fields**

* First Name (required)

* Last Name (required)

* Email (required)

* Phone (optional)

* SMS consent checkbox (if phone is collected)

* Who are you planning care for? — Myself / Parent / Spouse / Other (dropdown, required)

* How soon might care be needed? — Just researching / Within 1–3 years / Within 12 months / Immediately (dropdown, required)

**Why these were added:** Without at least a timeline question, every guide lead would score exactly 10 points (source score only) and always classify cold. Adding timeline and care-for allows guide leads with genuine urgency to score into warm or even hot range, which matches reality—some people downloading a guide are in active crisis.

### **Guide Delivery**

* Thank-you page displayed after submission with a download link or embedded PDF

* Follow-up email sent immediately with the PDF attached or linked

* Guide title: “The Assisted Living Planning Guide”

### **Guide Content Outline**

The guide itself should cover: signs it may be time for assisted living, types of care and what they mean, common family mistakes, cost basics by state, ways families pay for care, and next steps / planning tips. Guide creation is out of scope for the freelancer but this outline is provided for context.

# **H. Lead Scoring Model**

| Quiz lead | **\+20** |
| Calculator lead | **\+30** |

## **H9. Score Range Validation**

Before setting classification thresholds, here are the actual possible score ranges per funnel. This ensures the thresholds work fairly across all entry points.

| Funnel | Minimum Score | Maximum Score | Typical Range |
| :---- | :---- | :---- | :---- |
| **Guide** | 15 (source \+ timeline lowest) | 50 (source \+ timeline max) | 15–50 |
| **Quiz** | 40 (all minimums) | 225 (all maximums stacked) | 50–150 |
| **Calculator** | 50 (all minimums) | 155 (all maximums) | 55–120 |

**Note:** Guide leads now range 15–50 because they collect timeline data. Most guide leads will still fall in the cold range, but a guide lead with “Immediately” timeline will score 50 (warm), which is correct behavior—that person needs attention despite entering via a low-intent funnel.

# **I. Classification Thresholds & Rules**

| Classification | Score | Automation Actions |
| :---- | :---- | :---- |
| **Hot** | Over 70 | 1\. Set lead\_classification \= Hot 2\. Remove stage\_cold and stage\_warm tags 3\. Add stage\_hot tag 4\. Remove from any active cold/warm nurture 5\. Add nurture\_hot tag 6\. Enter hot lead workflow |
| **Warm** | 40–70 | 1\. Set lead\_classification \= Warm 2\. Remove stage\_cold and stage\_hot tags 3\. Add stage\_warm tag 4\. Remove from any active cold nurture 5\. Add nurture\_warm tag 6\. Enter warm nurture sequence |
| **Cold** | Under 40 | 1\. Set lead\_classification \= Cold 2\. Remove stage\_warm and stage\_hot tags 3\. Add stage\_cold tag 4\. Add nurture\_cold tag 5\. Enter cold nurture sequence |

**Re-classification rule:** If a contact later enters a second funnel (e.g., guide lead takes quiz), the workflow should update the contact record with the new data, re-calculate the score, and re-classify. The contact should not be duplicated. Match on email address.

# **J. Workflow Specifications**

## **J1. Quiz Intake Workflow**

**Trigger:** Quiz survey submitted

Actions in order:

1. Create or update contact (match on email)

2. Add tags: source\_quiz, quiz\_lead

3. Set care relationship tag based on care\_for response (parent\_care, self\_care, spouse\_care, other\_care)

4. Populate all custom fields from survey answers

5. Calculate lead\_score using scoring model

6. Set lead\_classification based on score thresholds

7. Apply classification tag (stage\_hot / stage\_warm / stage\_cold)

8. Remove any conflicting classification or nurture tags

9. Route: Hot → hot workflow, Warm → warm nurture, Cold → cold nurture

## **J2. Calculator Intake Workflow**

**Trigger:** Calculator form submitted

Actions in order:

10. Create or update contact (match on email)

11. Add tags: source\_calculator, calculator\_lead

12. Set care relationship tag

13. Populate all custom fields

14. Calculate lead\_score (includes level\_of\_care and familiarity scores from calculator-specific fields)

15. Set lead\_classification

16. Apply classification tag, remove conflicting tags

17. Display calculator results page with cost range

18. Route: Hot → hot workflow, Warm → warm nurture, Cold → cold nurture

## **J3. Guide Intake Workflow**

**Trigger:** Guide opt-in form submitted

Actions in order:

19. Create or update contact (match on email)

20. Add tags: source\_guide, guide\_lead

21. Set care relationship tag from care\_for dropdown

22. Populate custom fields (care\_for, care\_timeline)

23. Calculate lead\_score (source \+ timeline; limited data is expected)

24. Set lead\_classification

25. Apply classification tag

26. Send guide delivery email with PDF link

27. Route to appropriate nurture based on classification

## **J4. Hot Lead Workflow**

**Trigger:** Contact classified as Hot from any intake workflow

Actions in order:

28. Send immediate email: “Your next best step” with booking link

29. If sms\_consent \= true, send SMS with booking link (wait 10 minutes after email)

30. Internal notification: email or GHL notification to team with lead details, score, and quiz/calculator answers

31. Create opportunity in pipeline (if pipeline is set up)

32. Wait 24 hours

33. If no booking: send follow-up email \#2

34. Wait 48 hours

35. If still no booking: send email \#3, then move to warm nurture

## **J5. Warm Nurture Workflow**

**Trigger:** Contact classified as Warm, or Hot lead that did not book after 5 days

See Section K for the 5-email sequence with timing.

## **J6. Cold Nurture Workflow**

**Trigger:** Contact classified as Cold

See Section K for the 5-email sequence with timing.

## **J7. Re-Engagement Workflow**

**Trigger:** Contact completes nurture sequence without converting

Actions:

36. Wait 30 days after last nurture email

37. Send re-engagement email: “Still thinking about care options?”

38. If they click: push toward quiz or calculator (whichever they haven’t done)

39. If no engagement after 60 and 90 days: send one more email each, then move to newsletter-only list

# **K. Email Nurture Sequences**

All emails should be written in a warm, educational, non-salesy tone. The brand voice is “trusted advisor helping families navigate senior care.” Subject lines are suggestions; final copy should be reviewed before loading.

## **K1. Cold Nurture Sequence (5 Emails)**

Audience: early-stage, low urgency. Goal: build trust and move toward quiz or calculator.

### **Day 0: “Your Assisted Living Planning Guide”**

Deliver the guide (if from guide funnel) or thank them for completing the quiz/calculator. Reinforce that planning early is smart, not premature.

### **Day 3: “The biggest mistake families make with aging parents”**

Educational value. Address the common mistake of waiting until a crisis. No CTA other than “read more on our blog.”

### **Day 7: “How families actually pay for assisted living”**

Introduce the cost conversation gently. Mention savings, LTC insurance, Medicaid, VA benefits. Light CTA to calculator.

### **Day 14: “Is it time? 5 signs your parent may need more help”**

Self-assessment content. Let them see themselves in the signs. CTA to quiz if they haven’t taken it.

### **Day 21: “What to do when you’re not sure what to do”**

Empathy-driven. Acknowledge the overwhelm. Position True North as a resource. CTA to book a free planning call or take the quiz.

## **K2. Warm Nurture Sequence (5 Emails)**

Audience: moderate urgency, evaluating options. Goal: move toward calculator or consultation.

### **Day 0: “What your answers tell us about your situation”**

Interpret their quiz/calculator results. Personalize if possible (use merge fields for timeline, independence level). Reinforce next steps.

### **Day 2: “How families know it’s time to start comparing options”**

Decision framework content. When does “researching” become “it’s time to act”? Practical, not pushy.

### **Day 5: “What assisted living actually costs in Georgia”**

Concrete cost data. If they haven’t used the calculator, CTA to calculator. If they have, offer to discuss their specific situation.

### **Day 9: “The conversation most families put off too long”**

How to talk to a parent about assisted living. Emotional, practical. Story-driven. Builds trust.

### **Day 14: “Would it help to talk this through with someone?”**

Soft booking CTA. Position it as “a free 15-minute planning conversation, not a sales call.” Include booking link.

## **K3. Hot Lead Follow-Up (3 Emails)**

Audience: high urgency, ready to act. Goal: get them to book a consultation. Tone should be helpful and prompt without being aggressive.

### **Day 0 (immediate): “Your next best step”**

Acknowledge urgency. Based on their answers, a conversation would be the most helpful thing. Clear booking link. Brief, direct, warm.

### **Day 1: “Families in your situation often ask these 3 questions”**

Preemptively answer top questions (cost, timeline, how to talk to parent). Removes friction. Includes booking link again.

### **Day 3: “We’re here when you’re ready”**

Lower-pressure follow-up. Offer alternatives: call, email reply, or download a guide. If no action, move to warm nurture.

## **K4. Optional SMS for Hot Leads**

Only send if sms\_consent \= true. Send 10 minutes after the first email.

Template: “Hi \[First Name\], thanks for completing the assessment. Based on your answers, it may help to speak with someone. Here’s a link to book a free 15-min call: \[link\] — True North Senior Care”

**Compliance note:** Include business name in every SMS. Include opt-out instructions (“Reply STOP to unsubscribe”) in the first message. Do not send SMS before 8am or after 9pm local time.

# **L. Booking & Handoff Logic**

Hot leads should be moved to a direct next step as quickly as possible. Preferred flow:

40. Hot lead completes quiz or calculator

41. Results/thank-you page prominently displays booking CTA

42. Simultaneously, hot workflow sends email \+ optional SMS with booking link

43. Booking link goes to GHL calendar or Calendly page (/book)

44. Pre-call questionnaire auto-sent upon booking (5 short questions: what prompted you to look into care? what’s your biggest concern? have you toured any facilities? what’s your timeline? anything else we should know?)

45. When booking is confirmed: add tag booked\_consultation, remove from nurture, notify internal team

46. Internal team receives: contact name, phone, email, all custom field data, lead score, quiz/calculator answers, and pre-call questionnaire responses

If booking is not ready in v1, use a “next-step form” or questionnaire as a bridge, then route to manual follow-up.

# **M. Duplicate & Multi-Funnel Handling**

This was not in the original spec and is important to define.

**Match key:** Email address. All intake workflows should use “create or update” logic, not “create new.”

**Multi-funnel behavior:** If a contact enters through the guide and later takes the quiz, the workflow should: update all custom fields with the new (richer) data, add the new source tag (source\_quiz) alongside the existing one (source\_guide), re-calculate lead\_score using all available data, re-classify and re-route if classification changed.

**Score always uses best available data:** If a field was populated by a previous funnel and the new funnel doesn’t collect that field, keep the existing value. Only overwrite if the new funnel provides a response for that field.

**Nurture conflict resolution:** If a contact is mid-sequence in cold nurture and gets re-classified to warm, remove them from cold nurture and start warm nurture from the beginning.
